# 任务类别删除密码保护功能

## 功能概述

为防止误删除重要的任务类别，系统为9个核心任务类别添加了密码保护机制。删除这些类别时需要先验证管理员密码，确保只有授权用户才能执行删除操作。

## 保护的任务类别

以下9个任务类别受到密码保护：

1. **TC001 - 市场配合** - 市场配合相关任务
2. **TC002 - 项目执行** - 项目执行相关任务
3. **TC003 - 产品研发** - 产品研发相关任务
4. **TC004 - 科研任务** - 科研任务相关工作
5. **TC005 - 改造服务** - 改造服务相关任务
6. **TC006 - 内部会议与培训** - 内部会议与培训活动
7. **TC007 - 行政与党建** - 行政与党建相关工作
8. **TC008 - 差旅任务** - 差旅任务相关工作
9. **TC009 - 其他任务** - 其他未分类任务

## 技术实现

### 核心代码逻辑

#### 1. 重要类别识别
```typescript
// 重要任务类别列表（需要密码验证）
const importantTaskClassCodes = ['TC001', 'TC002', 'TC003', 'TC004', 'TC005', 'TC006', 'TC007', 'TC008', 'TC009'];
```

#### 2. 删除流程修改
```typescript
const handleDeleteTaskClass = (id: string) => {
  // 检查任务类别是否存在
  const taskClass = taskClasses.find(tc => tc.id === id);

  // 如果是重要类别，启动密码验证流程
  if (importantTaskClassCodes.includes(taskClass.code)) {
    setTaskClassToDelete(taskClass);
    setShowDeletePasswordPrompt(true);
    return;
  }

  // 非重要类别直接删除
  // ... 原有删除逻辑
};
```

#### 3. 密码验证逻辑
```typescript
const handlePasswordVerification = () => {
  // 使用 'admin' 作为删除验证密码
  if (deletePassword === 'admin') {
    if (taskClassToDelete) {
      dataService.deleteTaskClass(taskClassToDelete.id);
      loadData();
      showMessage('success', '任务类别删除成功');
    }
    // 清理状态
    setShowDeletePasswordPrompt(false);
    setDeletePassword('');
    setTaskClassToDelete(null);
  } else {
    showMessage('error', '密码错误，删除已取消');
    // 清理状态
    setShowDeletePasswordPrompt(false);
    setDeletePassword('');
    setTaskClassToDelete(null);
  }
};
```

#### 4. 状态管理
```typescript
const [showDeletePasswordPrompt, setShowDeletePasswordPrompt] = useState(false);
const [deletePassword, setDeletePassword] = useState('');
const [taskClassToDelete, setTaskClassToDelete] = useState<TaskClass | null>(null);
```

## 使用说明

### 删除重要类别流程

1. **点击删除按钮**
   - 进入设置 → 任务类别管理
   - 找到要删除的重要任务类别
   - 点击红色删除按钮（🗑️）

2. **输入密码验证**
   - 系统弹出密码验证对话框
   - 显示要删除的类别名称和编码
   - 输入管理员密码（默认：`admin`）

3. **确认删除**
   - 点击"确认删除"按钮
   - 系统验证密码正确后执行删除
   - 显示删除成功提示

4. **取消删除**
   - 点击"取消"按钮
   - 或直接关闭对话框
   - 系统清理输入的密码

### 密码验证失败

- **错误提示**: 显示"密码错误，删除已取消"
- **状态清理**: 自动清空密码和对话框
- **数据安全**: 不会执行删除操作

### 密码验证成功

- **执行删除**: 正常删除任务类别及其所有分类
- **成功提示**: 显示"任务类别删除成功"
- **状态清理**: 自动关闭对话框并清空状态
- **数据刷新**: 重新加载数据确保界面同步

## 安全性设计

### 密码保护策略

1. **单一密码**: 使用统一的管理员密码（`admin`）
2. **前端验证**: 密码验证在前端完成（简化实现）
3. **状态清理**: 验证后自动清理敏感信息
4. **不可绕过**: 重要类别删除必须经过密码验证

### 安全建议

#### 生产环境改进
1. **后端验证**: 将密码验证移到后端
2. **用户相关**: 使用当前用户的密码验证
3. **日志记录**: 记录密码验证尝试和删除操作
4. **密码强度**: 允许修改删除验证密码
5. **失败锁定**: 多次失败后锁定账户

#### 当前限制
- 密码是硬编码的 `admin`
- 密码验证在前端完成
- 没有失败次数限制
- 没有操作日志记录

## 问题排查

### Q: 删除按钮点击后没有反应？
**A**: 请检查：
1. 是否使用管理员或班组长账户登录
2. 类别是否为重要类别（TC001-TC009）
3. 浏览器控制台是否有错误

### Q: 密码验证对话框不显示？
**A**: 请检查：
1. 任务类别是否存在
2. 网络请求是否正常
3. JavaScript是否启用

### Q: 输入正确密码后仍然提示错误？
**A**: 请检查：
1. 密码是否输入正确（区分大小写）
2. 密码是否为："admin"
3. 是否有额外的空格

### Q: 如何修改删除验证密码？
**A**: 当前实现中密码是硬编码的 "admin"。要修改密码需要：
1. 找到 `handlePasswordVerification` 函数
2. 修改密码比较逻辑
3. 重新部署应用

**生产环境建议**:
- 使用用户自己的密码验证
- 将验证逻辑移到后端
- 添加配置项管理删除密码
