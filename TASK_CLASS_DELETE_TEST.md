### 2. 删除功能测试

#### 2.1 无依赖类别删除
1. 选择一个任务数量为0的类别
2. 点击删除按钮
3. 观察确认对话框

**对话框内容应包含**:
```
确定要删除任务类别"[类别名]"吗？

这将会同时删除该类别下的所有分类，
且无法恢复。

[确定] [取消]
```

4. 点击"确定"

**预期结果**:
- ✅ 类别从列表中消失
- ✅ 显示成功提示: "任务类别删除成功"
- ✅ 刷新页面后类别仍被删除

#### 2.2 有依赖类别删除
1. 选择一个任务数量大于0的类别
2. 点击删除按钮
3. 观察详细警告对话框

**对话框内容应包含**:
```
⚠️ 警告：删除任务类别"[类别名]"可能会影响以下内容：

• 该类别下有 X 个任务
• 删除后，这些任务将无法正常显示
• 相关的任务分类也将被删除

建议：
1. 先将任务移动到其他类别，或
2. 删除这些任务后再删除类别

确定要继续删除吗？

[确定] [取消]
```

4. 点击"确定"测试强制删除

**预期结果**:
- ✅ 类别被删除
- ✅ 显示成功提示
- ✅ 依赖的任务受影响（显示异常）

5. 重复步骤1-3，点击"取消"

**预期结果**:
- ✅ 对话框关闭
- ✅ 类别保留在列表中
- ✅ 无任何变化

## 常见问题排查

### 问题1: 删除按钮不显示
**可能原因**: 用户权限不足
**解决方案**: 使用管理员或班组长账户登录

### 问题2: 确认对话框不显示
**可能原因**: 浏览器阻止弹窗
**解决方案**: 允许该网站弹窗，或使用其他浏览器

### 问题3: 任务数量显示错误
**可能原因**: 数据同步问题
**解决方案**: 刷新页面，或重新登录

### 问题4: 删除后任务消失
**可能原因**: 任务被意外删除
**解决方案**: 检查是否误删，从备份恢复

### 问题5: 权限控制不生效
**可能原因**: 缓存问题
**解决方案**: 清除浏览器缓存，重新登录
