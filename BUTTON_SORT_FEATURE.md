# 按钮排序功能说明

## 功能概述

为任务分类管理增加了通过按钮点击调整分类顺序的功能，作为拖拽排序的补充，提供更精确的顺序控制方式。

## 主要特性

### 1. 上移/下移按钮
- **直观操作**: 点击按钮即可调整分类顺序
- **精确控制**: 每次移动一个位置，精确可控
- **即时反馈**: 操作后立即显示排序结果
- **边界处理**: 首位无上移，末位无下移

### 2. 按钮状态
- **可用状态**:
  - 上移按钮：非首位时可用，显示深灰色
  - 下移按钮：非末位时可用，显示深灰色
- **禁用状态**:
  - 上移按钮：首位禁用，显示浅灰色
  - 下移按钮：末位禁用，显示浅灰色
- **悬停效果**: 可用按钮悬停时背景变浅

### 3. 与拖拽功能结合
- **双重排序方式**: 支持按钮排序和拖拽排序
- **数据同步**: 两种方式调整的顺序实时同步
- **统一体验**: 排序结果在所有界面保持一致

## 技术实现

### 1. 新增处理函数

#### 上移函数
```typescript
const moveCategoryUp = (taskClassCode: string, categoryName: string) => {
  const categories = taskCategories[taskClassCode];
  const currentIndex = categories.indexOf(categoryName);
  if (currentIndex <= 0) return; // Already at the top

  const newOrder = [...categories];
  // Swap with previous element
  [newOrder[currentIndex], newOrder[currentIndex - 1]] =
  [newOrder[currentIndex - 1], newOrder[currentIndex]];

  dataService.reorderTaskCategories(taskClassCode, newOrder);
  setTaskCategories(dataService.getTaskCategories());
  showMessage('success', '分类排序已更新');
};
```

#### 下移函数
```typescript
const moveCategoryDown = (taskClassCode: string, categoryName: string) => {
  const categories = taskCategories[taskClassCode];
  const currentIndex = categories.indexOf(categoryName);
  if (currentIndex === -1 || currentIndex >= categories.length - 1) return; // Already at the bottom

  const newOrder = [...categories];
  // Swap with next element
  [newOrder[currentIndex], newOrder[currentIndex + 1]] =
  [newOrder[currentIndex + 1], newOrder[currentIndex]];

  dataService.reorderTaskCategories(taskClassCode, newOrder);
  setTaskCategories(dataService.getTaskCategories());
  showMessage('success', '分类排序已更新');
};
```

## 使用说明

### 基本操作

#### 1. 上移分类
1. 悬停在分类项上，显示操作按钮
2. 点击上移按钮（⬆️图标）
3. 分类向上移动一个位置
4. 显示"分类排序已更新"提示

#### 2. 下移分类
1. 悬停在分类项上，显示操作按钮
2. 点击下移按钮（⬇️图标）
3. 分类向下移动一个位置
4. 显示"分类排序已更新"提示

#### 3. 边界限制
- **首位分类**: 上移按钮禁用（灰色），无法上移
- **末位分类**: 下移按钮禁用（灰色），无法下移
- **中间分类**: 两个按钮都可用

### 高级操作

#### 1. 精确排序
- 多次点击上移/下移按钮
- 精确控制分类位置
- 适合需要细微调整的场景

#### 2. 结合拖拽使用
- 拖拽用于大幅调整位置
- 按钮用于精确微调
- 提高排序效率

#### 3. 批量排序
1. 确定目标顺序
2. 从第一个分类开始
3. 依次点击下移按钮调整
4. 或从最后一个开始，依次点击上移按钮

## 对比拖拽排序

| 特性 | 按钮排序 | 拖拽排序 |
|------|----------|----------|
| **操作方式** | 点击按钮 | 拖拽移动 |
| **精确度** | 高（一次移动一位） | 中（依赖拖拽位置） |
| **学习成本** | 低（直观按钮） | 中（需要拖拽技巧） |
| **适用场景** | 精确微调 | 大幅调整位置 |
| **速度** | 较慢 | 较快 |
| **可访问性** | 好（键盘可操作） | 差（需要鼠标） |
| **移动端支持** | 好 | 差 |

## 故障排除

### 问题1: 按钮点击无反应
**可能原因**:
- 已经是首位/末位，按钮被禁用
- 用户权限不足
- JavaScript错误

**解决方案**:
- 检查按钮状态（灰色表示禁用）
- 确认使用的是管理员/班组长账户
- 查看浏览器控制台错误

### 问题2: 排序结果未保存
**可能原因**:
- 浏览器localStorage被禁用
- 网络问题（虽然这是本地存储）

**解决方案**:
- 检查浏览器是否允许localStorage
- 刷新页面验证排序是否保持

### 问题3: 排序与任务管理不同步
**可能原因**:
- 页面缓存问题
- 数据未正确保存

**解决方案**:
- 刷新任务管理页面
- 重新登录系统
