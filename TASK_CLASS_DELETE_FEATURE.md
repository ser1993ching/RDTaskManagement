# 任务类别删除功能说明

## 功能概述

本次优化为任务分类管理增加了完整的任务类别删除功能，包括依赖检查、安全确认和视觉提示。

## 主要特性

### 1. 依赖检查
- **任务数量检查**: 自动检测是否有任务依赖该任务类别
- **分类统计**: 显示该类别下的分类数量
- **安全评估**: 根据依赖情况提供安全删除建议

### 2. 智能确认对话框
- **无依赖类别**: 显示简单的删除确认
- **有依赖类别**: 显示详细警告信息，包括：
  - 依赖的任务数量
  - 潜在影响说明
  - 建议操作步骤

### 3. 视觉提示
- **颜色编码**:
  - 🟢 绿色：无任务依赖，可安全删除
  - 🟠 橙色：有任务依赖，删除需谨慎
- **图标状态**:
  - 蓝色圆点：分类数量
  - 绿色圆点：无任务依赖
  - 橙色圆点：有任务依赖
- **悬停提示**:
  - 删除按钮显示当前依赖状态
  - 编辑按钮显示操作提示

### 4. 软删除机制
- **数据安全**: 使用`is_deleted`标记，而不是物理删除
- **可恢复性**: 删除后可以通过数据库恢复
- **数据完整性**: 保持历史数据完整性

## 技术实现

### 1. 新增方法

#### dataService.checkTaskClassUsage()
```typescript
checkTaskClassUsage(id: string): {
  hasTasks: boolean;    // 是否有任务依赖
  taskCount: number;    // 任务数量
  taskClassCode: string // 任务类别代码
}
```

**功能**:
- 检查指定任务类别是否有任务依赖
- 统计依赖的任务数量
- 返回任务类别代码用于进一步处理

### 2. 增强删除逻辑

#### Settings.handleDeleteTaskClass()
```typescript
const handleDeleteTaskClass = (id: string) => {
  // 1. 检查依赖
  const usage = dataService.checkTaskClassUsage(id);
  const taskClass = taskClasses.find(tc => tc.id === id);

  // 2. 生成确认消息
  let confirmMessage = '';
  if (usage.hasTasks) {
    // 有依赖的详细警告
    confirmMessage = `⚠️ 警告：删除任务类别"${taskClass.name}"可能会影响以下内容...\n\n...`;
  } else {
    // 无依赖的简单确认
    confirmMessage = `确定要删除任务类别"${taskClass.name}"吗？\n\n...`;
  }

  // 3. 执行删除
  if (confirm(confirmMessage)) {
    dataService.deleteTaskClass(id);
    loadData();
    showMessage('success', '任务类别删除成功');
  }
};
```

## 使用说明

### 删除任务类别

#### 步骤1: 访问任务类别管理
1. 登录系统（管理员或班组长权限）
2. 进入"设置" → "任务类别管理"

#### 步骤2: 选择要删除的类别
1. 查看任务类别列表
2. 观察状态指示器：
   - 🟢 绿色：可安全删除
   - 🟠 橙色：删除需谨慎

#### 步骤3: 执行删除
1. 点击删除按钮（🗑️图标）
2. 查看确认对话框：
   - **无依赖**: 简单的删除确认
   - **有依赖**: 详细警告和建议
3. 点击"确定"继续，或"取消"放弃

#### 步骤4: 确认结果
- 删除成功后显示绿色成功提示
- 任务类别从列表中消失
- 刷新页面验证删除结果

### 安全建议

#### 可安全删除的情况
- ✅ 该类别下没有任何任务
- ✅ 所有任务已移动到其他类别
- ✅ 所有任务已被删除

#### 需谨慎删除的情况
- ⚠️ 该类别下有活跃任务
- ⚠️ 该类别下有未完成的任务
- ⚠️ 该类别是系统默认类别

#### 删除前的准备工作
1. **检查任务列表**:
   - 进入任务管理
   - 查看该类别下的所有任务
   - 评估任务重要性

2. **迁移任务**:
   - 将任务移动到其他合适的类别
   - 或删除不需要的任务

3. **备份数据**:
   - 导出相关任务数据
   - 记录任务类别信息

## 注意事项

### 数据安全
1. **软删除**: 使用`is_deleted`标记，数据可恢复
2. **历史保留**: 删除后的类别仍保留在数据库中
3. **关联处理**: 删除类别不会自动删除任务，但会影响显示

### 业务影响
1. **任务显示**: 删除类别后，依赖的任务可能无法正常显示
2. **筛选功能**: 任务筛选可能受影响
3. **统计报表**: 数据统计可能不准确

### 最佳实践
1. **定期清理**: 定期检查和清理无用的任务类别
2. **任务迁移**: 删除前先迁移或删除相关任务
3. **团队沟通**: 删除前与团队成员沟通
4. **数据备份**: 重要数据提前备份

## 故障排除

### 问题1: 无法删除任务类别
**可能原因**:
- 用户权限不足（组员无法删除）
- 任务类别不存在
- 系统错误

**解决方案**:
- 使用管理员或班组长账户
- 刷新页面重试
- 检查浏览器控制台错误

### 问题2: 删除后任务消失
**可能原因**:
- 任务依赖的类别被删除
- 任务被意外删除

**解决方案**:
- 检查任务是否被标记为删除
- 从备份恢复任务类别
- 手动恢复任务数据

### 问题3: 删除确认对话框不显示
**可能原因**:
- 浏览器阻止了confirm对话框
- JavaScript错误

**解决方案**:
- 检查浏览器是否允许弹窗
- 查看浏览器控制台错误
- 刷新页面重试
