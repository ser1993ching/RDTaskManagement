# 任务分类管理功能说明

## 功能概述

本次优化为设置界面的任务分类管理增加了完整的二级分类管理功能，包括查看、编辑和删除操作。

## 功能特性

### 1. 任务类别选择
- 提供下拉选择器，可选择所有已配置的任务类别
- 选择后自动显示该类别下的所有二级分类

### 2. 二级分类展示
- 以卡片形式展示所有二级分类
- 显示分类总数统计
- 空分类时显示友好提示

### 3. 编辑分类名称
- 点击编辑按钮（铅笔图标）进入编辑模式
- 输入框自动聚焦，方便快速修改
- 支持保存和取消操作
- 保存时验证非空输入

### 4. 删除分类
- 点击删除按钮（垃圾桶图标）
- 删除前弹出确认对话框
- 防止误删操作

### 5. 添加新分类
- 在顶部输入框中添加新的二级分类
- 添加后自动刷新列表

## 操作说明

### 编辑分类名称
1. 在"任务分类管理"标签页中选择任务类别
2. 找到需要修改的分类项
3. 点击分类项右上角的编辑按钮（✏️）
4. 在弹出的输入框中修改分类名称
5. 点击"保存"按钮确认修改，或点击"取消"放弃修改

### 删除分类
1. 选择任务类别
2. 找到需要删除的分类项
3. 点击删除按钮（🗑️）
4. 在确认对话框中点击"确定"完成删除

### 添加新分类
1. 选择任务类别
2. 在"新增分类"输入框中输入分类名称
3. 点击"添加"按钮
4. 新分类将出现在列表中

## 技术实现

### 组件状态管理
- `editingCategory`: 当前正在编辑的分类名称
- `editingCategoryValue`: 编辑模式下的输入值

### 核心方法
#### Settings.tsx
- `handleUpdateTaskCategory()`: 更新分类名称
- `startEditingCategory()`: 开始编辑分类
- `cancelEditingCategory()`: 取消编辑

#### dataService.ts
- `updateTaskCategory()`: 数据层更新方法
  - 参数：`taskClassCode` (任务类别编码)
  - 参数：`oldCategoryName` (原分类名称)
  - 参数：`newCategoryName` (新分类名称)

### 数据流程
1. 用户点击编辑按钮 → 设置编辑状态
2. 用户修改名称并保存 → 调用handleUpdateTaskCategory
3. 组件调用dataService.updateTaskCategory
4. dataService更新localStorage中的数据
5. 组件重新加载数据并刷新UI

## 界面设计

### 卡片布局
- 采用响应式网格布局
- 桌面端：3列显示
- 平板端：2列显示
- 手机端：1列显示

### 编辑模式
- 原分类名称预填充到输入框
- 提供保存和取消按钮
- 按钮使用不同颜色区分（蓝色=保存，灰色=取消）

### 操作按钮
- 编辑按钮：蓝色，hover时背景变浅
- 删除按钮：红色，hover时背景变浅
- 按钮尺寸统一为14px图标

## 权限控制

- **管理员和班组长**：可以查看和操作所有功能
- **组员**：只能查看分类列表，无法编辑

## 注意事项

1. **数据一致性**：修改分类名称后，使用该分类的现有任务不会自动更新
2. **输入验证**：分类名称不能为空字符串
3. **实时保存**：修改后立即保存到localStorage，无需额外保存操作
4. **取消编辑**：取消编辑不会影响已保存的数据
